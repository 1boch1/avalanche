JointTraining

JointStrategy is a super class for all the joint training strategies.
class avalanche.training.strategies.joint_training.JointTraining(model: Module, optimizer: Optimizer, criterion, classifier_field: str = 'classifier', train_mb_size: int = 1, train_epochs: int = 1, test_mb_size: int = 1, device='cpu', plugins: Optional[Sequence[StrategyPlugin]] = None)[source]¶

JointStrategy is a super class for all the joint training strategies. This means that it is not a continual learning strategy but it can be used as an “offline” upper bound for them. This strategy takes in input an entire stream and learn from it one shot. It supports unique tasks (i.e. streams with a unique task label) or multiple tasks.

Parameters

  • model – PyTorch model.

  • optimizer – PyTorch optimizer.

  • criterion – loss function.

  • classifier_field – (optional) to specify the name of output layer.

  • train_mb_size – mini-batch size for training.

  • train_epochs – number of training epochs.

  • test_mb_size – mini-batch size for test.

  • device – PyTorch device to run the model.

  • plugins – (optional) list of StrategyPlugins.

create_task_layer(self, n_output_units: int, previous_task_layer=None)[source]¶

Creates a new task layer.

By default, this method will create a new Linear layer with n_output_units” output units. If “previous_task_layer” is None, the name of the classifier field is used to retrieve the amount of input features.

This method will also be used to create a new layer when expanding an existing task head.

This method can be overridden by the user so that a layer different from Linear can be created.

Parameters

  • n_output_units – The number of output units.

  • previous_task_layer – If not None, the previously created layer for the same task.

Returns

The new layer.

training_epoch(self, **kwargs)[source]¶

Training epoch. How does it work: 1. From each the n data loader (one for task) we load a mini-batch. 2. net forward with the right head accumlating gradients for all the n mini-batches. 3. Update the gradient. 1., 2. and 3. are repeated till all the data of each of the data loader have been processed at least once. If a data loader finishes his mini-batches, it starts again from the first mini-batch. This assumes that each task as roughly the same amount of data.

kwargs – named arguments eventually passed to this method.

Returns

None.
