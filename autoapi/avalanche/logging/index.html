class avalanche.logging.StrategyLogger[source]¶

Bases: StrategyCallbacks[None], abc.ABC

The base class for the strategy loggers.

Strategy loggers will receive events, under the form of callback calls, from the EvaluationPlugin carrying a reference to the strategy as well as the values emitted by the metrics.

Child classes can implement the desired callbacks. An alternative, simpler, mechanism exists: child classes may instead implement the log_metric method which will be invoked with each received metric value.

Implementing log_metric is not mutually exclusive with the callback implementation. Make sure, when implementing the callbacks, to call the proper super method.

Initialize self. See help(type(self)) for accurate signature.

class avalanche.logging.EvaluationPlugin(*metrics: Union[PluginMetric, Sequence[PluginMetric]], loggers: Union[StrategyLogger, Sequence[StrategyLogger]] = None, collect_all=True)[source]¶

Bases: avalanche.training.plugins.StrategyPlugin

An evaluation plugin that obtains relevant data from the training and testing loops of the strategy through callbacks.

This plugin updates the given metrics and logs them using the provided loggers.

Creates an instance of the evaluation plugin.

Parameters

  • metrics – The metrics to compute.

  • loggers – The loggers to be used to log the metric values.

  • (bool) (collect_curves) – enables the collection of the metric curves. If True self.metric_curves stores all the values of each curve in a dictionary. Please disable this if you log large values (embeddings, parameters) and you want to reduce memory usage.

avalanche.logging.accuracy_metrics(*, minibatch=False, epoch=False, epoch_running=False, task=False, train=None, test=None) → List[PluginMetric][source]¶

Helper method that can be used to obtain the desired set of metric.

Parameters

  • minibatch – If True, will return a metric able to log the minibatch accuracy.

  • epoch – If True, will return a metric able to log the epoch accuracy.

  • epoch_running – If True, will return a metric able to log the running epoch accuracy.

  • task – If True, will return a metric able to log the task accuracy. This metric applies to the test flow only. If the test parameter is False, an error will be raised.

  • train – If True, metrics will log values for the train flow. Defaults to None, which means that the per-metric default value will be used.

  • test – If True, metrics will log values for the test flow. Defaults to None, which means that the per-metric default value will be used.

Returns

A list of plugin metrics.
