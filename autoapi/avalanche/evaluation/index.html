class avalanche.evaluation.Metric[source]¶

Bases: Protocol[TResult]

Definition of a metric.

A metric exposes methods to reset the internal counters as well as a method used to retrieve the result.

The specific metric implementation exposes ways to update the metric state. Usually, standalone metrics like Sum, Mean, Accuracy, … expose an update method.

On the other hand, metrics that are to be used by using the EvaluationPlugin usually update their value by receiving events from the plugin (see the PluginMetric class for more details).

class avalanche.evaluation.PluginMetric[source]¶

Bases: Metric[TResult], StrategyCallbacks['MetricResult'], abc.ABC

A kind of metric that can be used by the EvaluationPlugin.

This class leaves the implementation of the result and reset methods to child classes while providing an empty implementation of the callbacks invoked by the EvaluationPlugin. Subclasses should implement the result, reset and the desired callbacks to compute the specific metric.

This class also provides a utility method, _next_x_position, which can be used to label each metric value with its appropriate “x” position in the plot.

Creates an instance of a plugin metric.

Child classes can safely invoke this (super) constructor as the first experience.
