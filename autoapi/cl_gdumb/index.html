GDumb

The GDumb strategy is built on top of the GDumb plugin and Naive strategy.
class cl_gdumb.GDumb(model: Module, classifier_field: str, optimizer: Optimizer, criterion: Module, mem_size: int, reinit_model_before_step: bool = True, reinit_function: object = None, train_mb_size: int = 1, train_epochs: int = 1, test_mb_size: int = None, device=None, evaluation_protocol: Optional[EvalProtocol] = None, plugins: Optional[Sequence[StrategySkeleton]] = None)¶

Bases: avalanche.training.strategies.Naive

The GDumb strategy is built on top of the GDumb plugin and Naive strategy. It retrain at each step the model with patterns taken solely from replay memory. It is up to the user logic to decide whether to reinitialize model at each step or proceed with finetuning. Original paper retrain from scratch at each step.

Creates an instance of the GDumb strategy.

Parameters

  • model – The model.

  • classifier_field – The name of the classifier field. Used when managing heads in Multi-Task scenarios.

  • optimizer – The optimizer to use.

  • criterion – The loss criterion to use.

  • mem_size – Replay memory size

  • reinit_model_before_step – if True, model will be reinitialized before each step according to reinit_function or uniformly if reinit_function is not provided. Default True.

  • reinit_function – a function which takes as input the model and return the model with weights initialized. Used only if reinit_model_before_step is True. Default None.

  • train_mb_size – The train minibatch size. Defaults to 1.

  • train_epochs – The number of training epochs. Defaults to 1.

  • test_mb_size – The test minibatch size. Defaults to 1.

  • device – The device to use. Defaults to None (cpu).

  • evaluation_protocol – The evaluation protocol. Defaults to None.

  • plugins – Plugins to be added (GDumb plugin will be added by default). Defaults to None.
